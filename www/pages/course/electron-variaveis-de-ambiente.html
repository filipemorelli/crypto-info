Electron - Variáveis ​​de ambiente

As variáveis ​​de ambiente controlam a configuração e o comportamento do aplicativo sem alterar o código. Certos comportamentos de elétrons são controlados por variáveis ​​de ambiente porque são inicializados anteriormente que os sinalizadores de linha de comando e o código do aplicativo.

Existem dois tipos de variáveis ​​de ambiente codificadas em elétrons - Variáveis ​​de produção e variáveis ​​de desenvolvimento.

Variáveis ​​de Produção

As seguintes variáveis ​​de ambiente destinam-se a ser utilizadas em tempo de execução em aplicativos de eletrônicos embalados.

S.No Variável e Descrição
1

GOOGLE_API_KEY

O eletrônon inclui uma chave de API codificada para fazer solicitações no webservice de geocodificação do Google. Como essa chave API está incluída em todas as versões do Electron, ela geralmente excede sua quota de uso.

Para contornar isso, você pode fornecer sua própria chave de API do Google no ambiente. Coloque o seguinte código em seu arquivo de processo principal, antes de abrir qualquer janela do navegador que faça pedidos de geocodificação -

process.env.GOOGLE_API_KEY = 'YOUR_KEY_HERE'
2

ELECTRON_RUN_AS_NODE

Inicia o processo como um processo normal do Node.js.
3

ELECTRON_FORCE_WINDOW_MENU_BAR (Somente Linux)

Não use a barra de menu global no Linux.

Variáveis ​​de desenvolvimento

As seguintes variáveis ​​de ambiente destinam-se principalmente a fins de desenvolvimento e depuração.

S.No Variável e Descrição
1

ELECTRON_ENABLE_LOGGING

Imprime o log interno do Chrome no console.
2

ELECTRON_ENABLE_STACK_DUMPING

Imprime o rastreamento da pilha no console quando o elétron falhar.
3

ELECTRON_DEFAULT_ERROR_MODE

Mostra o diálogo de falha do Windows quando o Electron falha.

Para definir qualquer uma dessas variáveis ​​de ambiente como verdadeiras, configure-as no seu console. Por exemplo, se você quiser habilitar o registro, use os seguintes comandos -

Para Windows

> set ELECTRON_ENABLE_LOGGING=true

Para Linux

$ export ELECTRON_ENABLE_LOGGING=true

Observe que você precisará configurar essas variáveis ​​de ambiente sempre que você reiniciar seu computador. Se você quiser evitar isso, adicione essas linhas aos seus arquivos .bashrc .
