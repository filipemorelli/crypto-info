Como o elétron funciona

O elétron leva um arquivo principal definido no seu arquivo package.json e o executa. Este arquivo principal cria janelas de aplicativos que contêm páginas web renderizadas e interação com a GUI nativa (interface gráfica do usuário) do seu Sistema Operacional.

À medida que você inicia um aplicativo usando o Electron, um processo principal é criado. Este processo principal é responsável pela interação com a GUI nativa do Sistema Operacional. Ele cria a GUI da sua aplicação.

Basta iniciar o processo principal não dá aos usuários do seu aplicativo nenhuma janela do aplicativo. Estes são criados pelo processo principal no arquivo principal, usando o módulo BrowserWindow . Cada janela do navegador executa seu próprio processo de renderização . O processo do processador leva um arquivo HTML que faz referência aos arquivos CSS usuais, arquivos JavaScript, imagens, etc. e o processa na janela.

O processo principal pode acessar a GUI nativa através de módulos disponíveis diretamente em Electron. A aplicação desktop pode acessar todos os módulos do nodejs, como o módulo do sistema de arquivos para manipulação de arquivos, solicitação para fazer chamadas HTTP, etc.

Diferença entre processos Main e Renderer

O processo principal cria páginas da web criando as instâncias do BrowserWindow . Cada instância do BrowserWindow executa a página da Web em seu próprio processo de renderização. Quando uma instância do BrowserWindow é destruída, o processo de renderização correspondente também é encerrado.

O processo principal gerencia todas as páginas da web e seus processos de renderização correspondentes. Cada processo de renderização é isolado e só se preocupa com a página da Web que está sendo executada nele.
